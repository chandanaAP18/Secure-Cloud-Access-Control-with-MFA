How It Works

A complete MFA and access-control workflow.

Each sign-in uses the registered email as identity, then moves through the website password and every configured factor before access is granted.

1

Email and website password

The user enters their registered email and the password created for this website. It is not the email account password.

2

Email OTP delivery

A one-time code is generated, hashed for storage, and sent to the user's email through SMTP.

3

Phone OTP, CAPTCHA, and image check

After email OTP, the flow continues through phone OTP, a generated CAPTCHA, and an image-selection challenge to add bot resistance and visual confirmation.

4

PIN and final routing

The user completes a PIN check, then is routed to the correct role-based dashboard with activity already logged.

System Outcome

Secure flow, visible stages, fewer blind spots.

The sequence reduces unauthorized access while giving administrators a record of events and users a clear, guided path through verification.

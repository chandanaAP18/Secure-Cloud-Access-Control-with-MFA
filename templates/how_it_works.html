How It Works

A complete MFA and access-control workflow.

Each sign-in uses the registered email as identity, then moves through the website password and every configured factor before access is granted.

1

Primary Authentication

The user enters their registered email and website-specific password. This initiates the multi-layered verification chain.

2

Email OTP Verification

A hashed one-time code is sent via SMTP. This confirms the user has access to their registered communication channel.

3

Bot Resistance Chain

A sequence of CAPTCHA, image selection, and text-based challenges prevents automated attacks and ensures human interaction.

4

Identity Knowledge

A private security question check ensures the user knows specific personal details previously configured for the account.

5

Secure PIN Check

A numeric Personal Identification Number provides an additional "something you know" layer to the authentication process.

6

Voice Biometrics

The user speaks a dynamic phrase. Voice pattern matching provides "something you are" biometric verification.

7

Authenticator TOTP

A time-based one-time password from a registered authenticator app (like Google Authenticator) adds mobile-based security.

8

Hardware Passkey

WebAuthn-based cryptographic proof using hardware security keys or device-level biometrics provides the highest level of assurance.

9

Final Secure Routing

Upon successful completion of all stages, the user is routed to their role-based dashboard with all activity logged for audit.

Secure flow, visible stages, fewer blind spots.
